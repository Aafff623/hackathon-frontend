# AgentCFO Demo Console — Design Direction

## 产品定位

AI Treasury Command Center for DAO payouts.

一个 DAO AI 财务指挥台 / AI CFO 控制台。不是传统 admin dashboard，而是让任何人都能看懂 Agent 正在如何分析、决策、执行付款的透明控制台。

## 页面信息层级

```
L1: 页面标题 + 定位声明
L2: Workflow Timeline（Agent 执行阶段总览）
L3: Payment Plan / Payout Board（付款计划详情）
L4: Risk Gate / Policy Check（风险检查结果）
L5: Human Approval（人类审批状态）
L6: Execution Result（执行结果）
L7: Audit Report / Settlement Receipt（审计报告）
```

## 视觉风格关键词

- 商务简报风
- 白底 / 浅灰底（`bg-white`, `bg-slate-50`）
- 克制蓝色强调（`text-blue-700`, `bg-blue-50`, `border-blue-200`）
- 清楚的信息卡片（`rounded-xl`, `border`, `shadow-sm`）
- 正式、干净、可信
- 拒绝：赛博紫、大面积渐变、AI 官网式发光、过度动效、廉价表格味

## 参考气质

- Google AI Studio 的清晰控制台感
- Lovable / Bolt.new 的轻量产品感
- MotionSites 的视觉节奏可参考，但不复制复杂动效

## Demo Story

```txt
一个 DAO 需要支付 3 位贡献者 + 1 个工具订阅。
AgentCFO 读取付款需求，生成 Payment Plan，执行 Risk Check，
拦截风险付款（Bob 不在白名单），请求人类审批，
模拟执行安全付款（Alice / Charlie / Data API），并输出 Audit Report。
```

## 核心区块

### 1. Workflow Timeline

展示 AgentCFO 的 7 个执行阶段：

```txt
Analyze input → Generate payment plan → Run risk checks → Request human approval → Execute payout → Generate audit report
```

表达：timeline / step cards / status rail。每步有状态标识。

### 2. Payment Plan / Payout Board

展示付款计划。3 位贡献者 + 1 个工具订阅：

| Recipient | Wallet | Amount | Token | Reason | Risk Level | Status |
|---|---|---|---|---|---|---|
| Alice | 0xAlice... | 20 | USDC | 技术文档 | Safe | Approved |
| Bob | 0xBob... | 15 | USDC | UI 设计 | **Blocked** | Rejected |
| Charlie | 0xCharlie... | 10 | USDC | AMA 主持 | Safe | Approved |
| Data API | 0xVendor... | 5 | USDC | 工具订阅 | Safe | Approved |

表达：payout board / settlement plan，不是普通 table。

### 3. Risk Gate / Policy Check

展示 4 项风险检查：

- Budget Check: ✅ Total 50 USDC within monthly budget
- Whitelist Check: ❌ Bob not in whitelist
- Single-Payment Limit: ✅ All within 25 USDC limit
- Duplicate Check: ✅ No duplicates

表达：Risk Gate / Policy Check / Risk Radar。突出"AI Agent 在可控边界内执行"。

### 4. Human Approval

展示审批状态：

- Safe payments → Pending approval → Approved
- Blocked payment (Bob) → Cannot execute

表达：说明 Agent 不会绕过人类审批。安全付款进入审批流程，被拦截的付款不能执行。

### 5. Execution Result

Mock execution（不接真实 Cobo）：

- Agent Wallet: 0xAgentWalletPlaceholder
- Network: Base Sepolia (placeholder)
- 3 successful transactions (Alice 20, Charlie 10, Data API 5)
- Mock tx hashes
- Timestamps

表达：明确标注 "Simulated" / "Mock Execution" / "Testnet Placeholder"。

### 6. Audit Report / Settlement Receipt

正式 settlement receipt：

- Approved payments: 3 items, 35 USDC
- Blocked payments: 1 item (Bob), 15 USDC
- Total plan: 4 items, 50 USDC
- Risk summary: Whitelist check blocked 1 recipient
- Execution summary: 3 simulated transactions on Base Sepolia
- Generated by: AgentCFO
- Mock execution note: "This is a simulation. No real blockchain transaction occurred."

表达：正式收据感，不是普通日志。

## Mock-First 数据流

```txt
lib/mock/
  ├── contribution-records.ts   → contributors + contributions + subscriptions
  ├── payment-plan.ts           → PaymentPlan (4 items, 50 USDC)
  ├── risk-check.ts             → RiskCheckResult (Bob blocked)
  ├── caw-execution.ts          → CAWExecution[] (3 success)
  ├── audit-report.ts           → AuditReport (summary)
  └── budget-rules.ts           → BudgetRule (policy)

lib/api/
  ├── types.ts                  → TypeScript interfaces
  ├── client.ts                 → DEMO_MODE + request()
  ├── payment.ts                → real adapter (mock-first)
  └── caw.ts                    → real adapter (mock-first)

lib/workflow/
  ├── demo-steps.ts             → 7 DemoStep definitions
  └── demo-state-machine.ts     → state machine types

page.tsx (Server Component)
  → imports mock data directly
  → passes data to presentation components
  → no fetch, no client state
```

## 本轮不做什么

- 不安装 HeroUI Pro
- 不运行 `hpsetup`
- 不配置 `hpmcp`
- 不安装 Aceternity / shadcn/ui / Framer Motion
- 不接真实 API
- 不接真实 Cobo Agentic Wallet
- 不做复杂交互状态机
- 不做 dark mode
- 不做响应式精细化适配（基础 responsive 即可）
- 不做动画
